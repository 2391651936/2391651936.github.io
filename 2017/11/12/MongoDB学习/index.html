二、MongoDB 学习

1、什么是 MongoDB

MongoDB 是由C++语言编写的，是一个基于分布式文件存储的开源数据库系统。

在高负载的情况下，添加更多的节点，可以保证服务器性能。

MongoDB 旨在为WEB应用提供可扩展的高性能数据存储解决方案。

MongoDB 将数据存储为一个文档，数据结构由键值(key=>value)对组成。MongoDB 文档类似于 JSON 对象。字段值可以包含其他文档，数组及文档数组。

2、MongoDB 基本概念

SQL 术语/概念 MongoDB 术语概念 解释/说明
database database 数据库
table collection 数据表库/集合
row document 数据库表/集合
column field 数据字段/域
index index 索引
table join 表连接，MongoDB 不支持
primary key primary key 主键，MongoDB 自动将 _id 设置为主键

5、文档

所有的数据都是一个文档 document

  1. 插入文档：
    db.COLLECTION_NAME.insert(document)

    例如：db.person.insert({name: "杨伟健", age: 23})

  2. update 更新文档：
    db.COLLECTION_NAME.update(query, update, options)

    参数说明：

    • query：update 查询条件，一个 Object 对象
    • update：update 的对象和一些更新的操作符，例如：{$set: {name: "张三"}}

    • options：

      • upsert：如果不存在的话，插入记录
      • multi：mongodb 默认是 false，只更新找到的第一条记录，如果这个参数为 true，就把按条件查出来多条记录全部更新。
      • writeConcern：抛出异常的级别

      例如：db.person.update({name: "李四"}, {$set: {age: 1}})

  3. save 更新文档：
    db.COLLECTION_NAME.save(obj)

    参数说明：

    • obj：要替换的文档，必须有 _id

  4. 删除文档：
    db.COLLECTION_NAME.remove(query, options)

    参数说明：

    • query：删除文档的条件

    • options：

      • juestOne：如果设为 true 或 1，则只删除一个文档，如果不设置该参数，或使用默认值 false，则删除所有匹配条件的文档。
      • writeConcern：抛出异常的级别

      例如：db.preson.remove({name: "张三"})

  5. 显示所有文档：
    db.COLLECTION_NAME.find()

  6. 查找满足条件中的第一个
    db.COLLECTION_NAME.findOne(condition)

    condition：一个 Object 对象

  7. 使用 $or 代表 or

6、条件操作符

条件 操作符

例如：

获取年龄小于20的 person：db.person.find({age: {$lt: 20}})

8、limit() 方法

查询前两条记录：db.person.find().limit(2)

9、skip() 方法

只显示第二条记录：db.person.find().skip(1).limit(1)

10、sort() 方法

在 MongoDB 中使用 sort() 方法对数据进行排序，sort() 方法可以通过参数指定排序的字段，并使用 1 和 -1 来指定排序的方式，其中 1 为升序排列，而 -1 是用于降序排列。

以年龄(age)升序排列：db.person.find().sort({age: 1})

11、索引

  1. 创建索引：
    db.collection.createIndex(keys, options)

    参数说明：

    • background(boolean): 建索引过程会阻塞其它数据库操作，background可指定以后台方式创建索引，即增加 “background” 可选参数。 “background” 默认值为false。
    • unique(boolean): 建立的索引是否唯一。指定为true创建唯一索引。默认值为false。
    • name(string): 索引的名称。如果未指定，MongoDB的通过连接索引的字段名和排序顺序生成一个索引名称。
    • sparse(boolean): 对文档中不存在的字段数据不启用索引；这个参数需要特别注意，如果设置为true的话，在索引字段中不会查询出不包含对应字段的文档.。默认值为 false.
    • expireAfterSeconds(integer): 指定一个以秒为单位的数值，完成 TTL设定，设定集合的生存时间。
    • v(index): 索引的版本号。默认的索引版本取决于mongod创建索引时运行的版本。
    • weights: (document): 索引权重值，数值在 1 到 99,999 之间，表示该索引相对于其他索引字段的得分权重。
    • default_language(string): 对于文本索引，该参数决定了停用词及词干和词器的规则的列表。 默认为英语
    • language_override(string): 对于文本索引，该参数指定了包含在文档中的字段名，语言覆盖默认的language，默认值为 language.
  2. 查看集合的索引
    db.collection.getIndexes()
  3. 查看集合索引大小
    db.collection.totalIndexSize()
  4. 删除集合所有索引
    db.collection.dropIndexes()
  5. 删除集合指定索引
    db.collection.dropIndex("索引名称")
