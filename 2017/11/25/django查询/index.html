2、Q 对象查询

仅靠单一的关键字参数查询已经很难满足查询要求，此时 django 为我们提供了 Q 查询：

1、Q 对象(from django.db.models import Q)可以对关键字参数进行封装，从而更好地应用多个查询，下面是例子：

2、可以组合使用&，|操作符，当一个操作符作用于两个对象，它产生一个新的Q对象。

Q(question__startwith="What") | Q(question__startwith="Who")

3、Q对象可以用~操作符放在前面标识否定，也可以允许否定与肯定形式的组合。

Q(question__startwith="What") | ~Q(question__startwith="Who")

4、Q对象可以与关键字参数查询一起使用，不过一定要把Q对象放在关键字参数查询的前面。
