Serializable 接口

什么是 java 序列化？

1、必须要实现 Serializable 接口

java 序列化技术可以将一个对象的状态写入一个 byte 流中，并且可以其他需要的地方把 byte 流里的数据读出来。

序列化 ID 有两种生成策略：

1. 固定的 1L
2. 随机生成一个不重复的 long 类型数据（实际上是使用 JDK 工具，根据类名、接口名、成员方法及属性等来生成）

序列化 ID 的作用：当存储的类的序列化 ID 与反序列 ID 不同的话，不能进行反序列化操作。

注意： 这里的输出，school 为空，是由于：静态变量是属于类的，而成员变量属于对象，在写数据时只能保存对象的非静态成员交量，不能保存任何的成员方法和静态的成员变量，而且保存的只是变量的值，对于变量的任何修饰符都不能保存。

所以在进行反序列化的时候，字段：school 才会得到 null 这个值。

3、transient 关键字

说明： transient 关键字的作用是：阻止实例中那些用此关键字声明的变量持久化；当对象被反序列化时(从源文件读取字节序列进行重构)，这样的实例变量值不会被持久化和恢复。
