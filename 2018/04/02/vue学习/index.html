一、什么是 vue ？

Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

以上内容来自 Vue 官方网站的解答。

我理解的 Vue ：

  1. 一个轻量级 MVVM 框架，在我了解中，这是最简单、易学的一个 MVVM 框架了。
  2. 数据驱动界面。
  3. 前端、后台，完全分离的形式，后台代码提供 data 接口，在前端的 Vue 界面中进行访问。
  4. 实现“模块化开发”，把一个页面差分成许多个模块（模块是可以复用的），在想要显示的界面上，把已经开发好的模块拼接上去，组成新的界面。

1. 路由跳转及设置

第一天碰到的坑，就是 vue-router，对于一个新手来说，真的不好理解，根本不知道组件是怎么拼接出来。

  1. router-view
    我对这个的理解，他是用来把 #/ 后面的路径所对应的组件拼接到当前界面，这里需要注意一下：，它只能寻找到所碰到的第一个/，在往后面的/就是拼接到第二级界面上的组件。
    比如说：/a/b/c (假设a、b、c都为正常路径，不会作为参数)
    • 那/a对应的就是App.vue中的router-view，/a进入a.vue中
    • 那/a/b对应的就是a.vue中的router-view， /a/b进入b.vue中
  2. router-link，路由跳转
    props：
    • to
      • 类型：string
      • required
        渲染路径，例如：<router-link to="home">Home</router>，渲染的结果为：<a href="home">Home</a>
      • 如果，当前路由有name的话，可以写成<router-link :to="{ name: 'home'}">User</router-link>
      • 带参数：<router-link :to="{ name: 'home', params: {id, '1'}}">User</router-link>，相当于：/home?id=1
        还有些别的参数，这里还没测试到，等用到在进行补充

2. 异步请求

在这里，我使用的是 vue-resource 这个库

以前使用的更多的是 jQuery，vue-resource 与 jQuery 比较的话，vue-resource 更加小巧，由于 jQuery 中包含更多的是对 dom 的操作，所以会更大些。

vue-response 标准语法，this.$http.get(‘url’, [data], [option]).then();

option 的参数

url string 请求的URL
method string 请求的HTTP方法，例如：’GET’, ‘POST’或其他HTTP方法
body Object,FormDatastring request body
params Object 请求的URL参数对象
headers Object request header
timeout number 单位为毫秒的请求超时时间 (0 表示无超时时间)
before function(request) 请求发送前的处理函数，类似于jQuery的beforeSend函数
progress function(event) ProgressEvent回调处理函数
credientials boolean 表示跨域请求时是否需要使用凭证
emulateHTTP boolean 发送PUT, PATCH, DELETE请求时以HTTP
emulateJSON boolean 将request body以application/x-www-form-urlencoded content type发送
