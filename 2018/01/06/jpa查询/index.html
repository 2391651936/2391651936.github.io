三、hibernate 时间处理

javax.persistence.Temporal 这个注解

@Temporal(TemporalType.DATE) 日期，不能自动生成，需要自己添加，对应 mysql 中的 date 字段

@Temporal(TemporalType.TIME) 时间， 不能自动生成，需要自己添加，对应 mysql 中的 time 字段

@Temporal(TemporalType.TIMESTAMP) 时间， 不能自动生成，需要自己添加，对应 mysql 中的 datetime 字段，_ 从数据库取出来的时候，是时间戳，需要转换。_

十、实体字段注解

@Column

可选

  • name：表示数据库表中该字段的名称，默认与属性名称一致。

  • nullable：表示该字段是否允许为null，默认为true。

  • unique：表示该字段是否是唯一表示，默认为false。

  • length：表示该字段的大小，仅对String类型的字段有效。

  • insertable：表示在ORM框架执行插入操作时，该字段是否应该出现在INSERT语句中，默认为true。

  • updateable：表示在ORM框架执行更新操作时，该字段是否应该出现在UPDATE语句中，默认为true，对于一经创建就不可以更改的字段，该属性非常有用。

columnDefinition：表示该字段在数据库中的实际类型，通常ORM框架可以根据属性类型自动判断数据库中字段的类型，但是对于Date类型无法判断是DATE, TIME, TIMESTAMP。此外，String类型默认映射成VARCHAR，如果将String类型映射到特定数据库的类型，需要用到此属性。

@Transient

可选

  • 表示该属性并非一个到数据表的字段的映射，ORM框架将忽略该属性。

@JoinColumn

可选

  • name：该字段的名称，由于@JoinColumn描述的是一个关联子字段，如ManyToOne，则默认的名称由关联的实体决定。

@OneToMany

可选

  • fetch：表示抓取策略，默认为FetchType.LAZY，因为关联的多个对象通常不必从数据库预先读取到内存。

  • cascade：表示级联操作策略，对于OneToMany类型的关联非常重要，通常该实体更新或删除时，其关联的试题也应当被更新或删除。

  • mappedBy：表示该字段对应外键表中的那个字段。

@ManyToOne

可选

  • optional：是否允许该字段为null，该属性应该根据数据表的外键约束来确定，默认为true。

  • fetch：表示抓取策略，默认为FetchType.EAGER。

  • cascade：表示默认的级联操作的策略，可以指定为ALL, PERSIST, MERGE, PEFRESH, REMOVE中的若干组合，默认为无级联操作。

  • targetEntity：表示该属性关联的试题类型，该属性通常不必指定，ORM框架根据属性类型自动判断targetEntity。

@OneToOne

可选

  • fetch：表示抓取策略，默认为FetchType.LAZY

  • cascade：表示级联操作策略

@ManyToMany

可选

  • targetEntity：表示多对多关联的另一个实体类的全名，例如：package.Book.class

  • mappedBy：表示多对多关联的另一个实体类的对应集合属性名称
