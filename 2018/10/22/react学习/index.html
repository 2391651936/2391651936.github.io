4、受控组件

在 HTML当中，像 <input> , <textarea> , 和 <select> 这类表单元素会维持自身状态，并根据用户输入进行更新。但在 React中，可变的状态通常保存在组件的状态属性中，并且只能用 setState() 方法进行更新。

向 3 的例子，就是一个受控组件

5、非受控组件

在大多数情况下，我们推荐使用受控组件来实现表单。在受控组件处理，如果让表单数据由DOM 处理时，替代方案为使用非受控组件。

在使用非受控组件的时候，需要使用到 ref 从 DOM 中获取表单值。

6、状态提升

使用 react 经常会遇到几个组件需要共用状态数据的情况。这种情况下，我们最好将这部分共享的状态提升至他们最近的父组件当中进行管理。

由于 react 的数据流是单项传递，也就是说，只能从父组件传递到子组件中，父组件定义的数据，只能自己访问，如果想让子组件也访问到，需要把父组件中的数据传递给子组件。

8、Context 上下文

Context 通过组件树提供了一个传递数据的方法，从而避免了在每一个层级手动的传递 props 属性。

Context 设计目的是为共享那些被认为对于一个组件树而言是“全局”的数据，例如当前认证的用户、主题或首选语言。

9、Fragments

React 中一个常见模式是为一个组件返回多个元素。Fragments 可以让你聚合一个子元素列表，并且不在 DOM 中增加额外节点。

10、错误边界

错误边界是用于捕获其子组件树 JavaScript 异常，记录错误并展示一个回退的 UI的 React 组件，而不是整个组件树的异常。错误组件在渲染期间，生命周期方法内，以及整个组件树构造函数内捕获错误。

11、高阶组件

高阶组件 HOC 是 react 中对组件逻辑进行重用的高级技术。但高阶组件本身并不是 React API。它只是一种模式，这种模式是由 react 自身的组合性质必然产生的。

具体而言：高阶组件就是一个函数，且该函数接受一个组件作为参数，并返回一个新的组件
