一、HTML 5 历史记录 API => BrowserRouter

属性：

  1. basename(string)：所有 Router 的基本 URL，如果应用程序从服务器上的子目录提供，则需要将其设置为子目录。正确格式的基础名称应该有一个主要的斜杠，但没有尾部斜线。例如：

  2. getUserConfirmation(func)：导航到此页面前执行的函数，默认使用 window.confirm，例如：

  3. forceRefresh(bool)：如果为 true，则路由器将在页面导航中使用全页刷新，而不是平常使用的局部刷新。

  4. keyLength(number)：设置 Route 传递给组件中 location.key 的值，默认为 6，点击同一个连接，location.key 的值都会发生改变。

六、Route 渲染方式

  • <Route component/>
  • <Route render/>
  • <Route children/>

虽然提供三种使用方式，但是你只能使用其中的一种；每种渲染方式都将通过相同的三个 route props

  • match
  • location
  • history

  1. component(组件)：仅当位置匹配时才呈现的 React 组件，他将与 route props 一起呈现，例如：

  2. render(func)：可以更方便的对组件进行包装，可以使用户高阶组件对 Route 进行再一次的包装，例如：

  3. children(func)：不理解。。。

  4. path(string)：url 路径
  5. exact(bool)：为 true 时，只有路径完全匹配才显示。
  6. strict(bool)：为 true 时，匹配尾部斜杠

七、switch 渲染被其包裹的 Route 或 Redirect 第一个匹配 location 的哪一个。

八、history 对象（props 中的 history 对象）

  1. length(number)：当前历史栈中存在的实体数量。
  2. action(stirng)：当前路由的 action (PUSH, REPLACE, or POP)
  3. location(object)：当前的 location
  4. push(func)：在历史栈中压入一个实体
  5. go(func)：通过n个条目移动历史堆栈中的指针
  6. goBack(func)：返回上一级 => go(-1)
  7. goForward(func)：相当于 => go(1)
  8. block(func)：但离开网页时，会触发一个 confirm。
