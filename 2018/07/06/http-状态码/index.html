http 状态码

发表于 2018-07-06 | 分类于 http

http 状态码以 3 位数字和原因短语组成，以下介绍常用的 14 个状态码。

2XX 成功

200 OK

表示从客户端发来的请求在服务器端被正常处理了。

在相应报文内，随状态码一起返回的信息会因 方法的不同 而发生改。

比如： 使用 GET 方法，对应请求资源的实体会作为相应返回；使用 HEAD 方法，对应请求资源的实体，不会随着报文首部作为相应返回。

204 No Content

代表服务器接收的请求已成功处理，但在返回的相应报文中不含实体的主题部分。另外，也不允许返回任何实体的主体。

比如： 当从浏览器发出请求处理后，返回 204 响应，那么浏览器像是的页面不发生更新。一般只在需要从客户端往服务器发送信息，而客户端不需要发生新信息内容的情况下使用。

206 Partial Content

该状态码表示客户端进行了范围请求，而服务器成功执行了这部分的 GEt 请求。

在 请求头 设置 Range: byte=[start]-[end] 代表想要下载的范围；在 响应头 设置 Accept-Ranges: bytes 、 Content-Type: [文件的 MIMETYPE] 、 Content-Disposition: 、 Content-Range: bytes [要下载的开始位置]-[结束位置]/[文件总大小]，并设置状态码为 206。

3XX 重定向

301 Moved Permanently

永久性重定向。请求资源已永久以以偶定到新位置。

例如： 在 chrome 上，有一个链接 http://127.0.0.1:8080/301 重定向到 www.baidu.com 上，那么，在没有清除缓存之前，第一个链接永远会跳转到第二个链接上，被称之为：永久重定向。

在 响应头 中 设置 Location: www.baidu.com 才会被重定向。

302 Found (已不推荐使用)

临时重定向。请求资源已被分配了新的 URI 希望用户本次能使用新的 URI 进行访问。和 301 Moved Permanently 状态码相似，但 302 状态码代表的资源不是被永久移动，只是临时性质的。换句话说，已移动的资源对应的 URI 未来还有可能发生改变。

例如： 在 chrome 上，有一个链接 http://127.0.0.1:8080/301 重定向到 www.baidu.com 上，在修改重定向之后，重启服务器，就会重新跳转到新的链接。

303 See Other

该状态码表示对象鱼请求对应的资源存在着另一个 URI 应使用 GET 方法重新定向获取的请求的资源。

303 状态码是对 302 状态码更加确切的描述(ps：我是这样认为的)。

比如： 当使用 POST 方法请求一个 URL 其执行后的处理结果时希望客户端能以 GET 方法重定向到另一个 URL 上去时，返回 303 状态码。

304 Not Modified

该状态码表示客户端发送附带条件的请求时，服务器允许请求访问资源，但因发生请求为满族条件的情况后，直接饭后 304 状态码返回。

送附带条件的请求： 是指采用 GET 方法的请求报文中包含：If-Match 、 If-Modified-Since 、 If-None-Match 、 If-Range 、 If-Unmodified-Since 中的任一首部。

307 Temporary Redirect

会以当前请求方法，去请求重定向后的链接。

例如： 以 POST 方法去请求 http://127.0.0.1:8080/307 时，被重定向到 http://127.0.0.1:8080/ 会以 POST 的方法去请求新的连接。

4XX 客户端错误

400 Bad Request

该状态码表示请求报文中存在语法错误。一般出现在：浏览器发送的参数格式不正确。

401 Unauthorized

该状态码表示发送的请求需要有通过 HTTP 认证(BASIC 认证、DIGEST 认证)的认证信息。

如果响应头返回一个 401 状态码，浏览器会弹出一个登录框，当用户填写好账号、密码后，会回送到当前 URL 中，并在求情头中添加 Authorization: Basic [账号:密码]的BASE64编码 由于 BASE64 编码不安全，所以会出现一些问题。认证成功之后，只有当浏览器关闭时才会删除登录状态。

403 Forbidden

该状态码表明对请求资源的访问被服务器拒绝了。服务器端没有必要给出拒绝的详细理由，但如果想作说明的话，可以在实体的主题部分对原因进行描述，这样就能让用户看到了。

404 Not Found

该状态码表明服务器上无法找到请求的资源。除此之外，也可以在服务器端拒绝请求且不想说明里有时使用。

5XX 服务器错误

500 Internal Server Error

该状态码表明服务器端在执行请求时发生了错误。也有可能是 web 应用存在的 bug 或某些临时的故障。说白了就是编码出现问题。

503 Service Unavailable

该状态码表明服务器暂时处于超负载或正在进行停机维护，现在无法处理请求。
